WI:SH Kiosk — 키오스크 접근성 개선 프로젝트

어르신과 장애인의 주문 경험을 개선하기 위해 손글씨 입력(Write It), 스캔(Scan), 처리(Handle)를 통합한 키오스크 시스템입니다. 스캐너(WIA) 기반 입력, QR/바코드 인식(ZXing, QRCoder), 그리고 학습된 AI 모델(PyTorch/ONNX)을 통해 필기 인식을 지원합니다.

GitHub 저장소 열기 빠른 시작

핵심 목표

  • 손글씨 기반 주문 입력과 접근성 강화
  • 스캔-인식-처리 전체 흐름의 자동화
  • 현장 배치에 필요한 운영 구성 제공

기술 스택

C# (.NET 8.0-windows)
WIA (스캐너)
ZXing
QRCoder
PyTorch → ONNX

저장소 구조 (요약)

  • KioskAI/ — ONNX 모델 로드/추론
  • wishKioskDID/ — DID 관련 모듈
  • wishKioskDIDReceive/ — 수신/알림 모듈
  • copy_these_files/ — 실행 폴더로 복사할 파일
  • trainModel.py — 모델 학습/변환 스크립트

주요 기능

  • 스캐너(WIA)로 입력된 용지를 자동 감지 및 이미지 전처리
  • ZXing을 통한 바코드/QR 인식, QRCoder로 QR 생성
  • ONNX 런타임 기반 손글씨 분류/인식(모델은 PyTorch로 학습)
  • 주문 수신/알림을 위한 보조 실행 모듈 구성
상세 기능은 저장소 커밋/README 기준이며, 변경될 수 있습니다.

시스템 구성

  1. 사용자 필기 → 스캐너(WIA)로 이미지 획득
  2. 이미지 전처리 → 바코드/QR 인식
  3. 모델 추론(ONNX)로 필기 내용 판별
  4. 주문 처리 및 DID 수신 모듈로 전달

설치 및 준비

사전 요구사항

  • Windows 10/11
  • Visual Studio 2022 (최신 .NET 워크로드)
  • .NET SDK 타깃: net8.0-windows
  • WIA 호환 스캐너 드라이버

다운로드

Git을 사용하거나 ZIP으로 내려받을 수 있습니다. 

git clone https://github.com/Team-ToyoTech/WISH-Kiosk.git
cd WISH-Kiosk

Visual Studio에서 열기

  1. wishKiosk.sln 솔루션을 엽니다.
  2. NuGet 패키지를 복원합니다(우클릭 → Restore).
  3. 솔루션 빌드(Ctrl+Shift+B).

실행 전 파일 복사

저장소의 copy_these_files 폴더 내용을 실행 파일 위치로 복사합니다.

  • onnx_model → \KioskAI\bin\Debug\net8.0-windows
  • sound.wav → \wishKioskDIDReceive\bin\Debug\net8.0-windows
위 경로는 README의 안내에 기반합니다. 실제 배포/릴리스 구성 시 폴더가 달라질 수 있습니다.

사용 방법

  1. 필요 모듈(예: KioskAI, wishKioskDIDReceive)을 시작합니다.
  2. 스캐너가 WIA 드라이버로 인식되는지 확인합니다.
  3. 주요 창에서 스캔을 수행하고 결과가 표시되는지 확인합니다.
정확한 실행 순서/옵션은 저장소 문서를 참조하십시오. 현재 공개된 정보로는 일부 단계가 불명확합니다.
  1. 스캔 버튼 → 문서 스캔
  2. 바코드/QR 감지 → 주문 식별
  3. 모델 추론 → 분류/인식 결과
  4. 주문 처리 → DID 수신/알림
  • 스캐너 인식 실패: WIA 드라이버 확인, 다른 USB 포트 사용
  • 패키지 오류: NuGet 패키지 복원/재설치
  • 모델 로드 실패: onnx_model 경로 재확인
